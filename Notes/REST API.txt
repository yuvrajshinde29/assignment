REST:

	REST is an architectural style that defines a set of rules for building scalable web services.


		Client server seperation:
			they communicate using HTTP but are independent.
			Ex- react app fetchs data from node.js REST API
			
		stateless-
			Every HTTP request must contain all the information the server neededs.
			server does not store any session info between requests.
			Benefit: Scalability and reliability.
			
		cacheble-
			Response should indicate if they can cached.
			Reduce server load and speed up response time
			
		uniform interface-
			Always interact with resources in a consistent, predictable way.
				Resource identification- Every thing you want has its own url
					Example: /api/users/5 → the user with ID 5.
					
				Manipulation via representaions- 
					you change resources by sending data in a standard format like JSON.
					
				Self-descriptive Messages-	
					The request itself tells what’s happening using HTTP verbs (GET, POST, PUT, DELETE) and status codes (200, 404, etc.)

	Resource:
		the data representation is called resource.
		
	1. Singleton and Collection resource-
			Ex-
				URL: /customers			// is a collection resource
				URL: /customers/{id}		// is a singleton resource

	2. Collection and sub-collection resources
		 sub-collection resource “accounts” of a particular “customer” -/customers/{customerId}/accounts
			/customers						//is a collection resource
			/customers/{id}/accounts		// is a sub-collection resource
	-----------------------
	URI :( Uniform Resource Identifiers )
		URI identifies a resource
		
	----------------------------
	Best Practices:
		#use nouns to represent resources
			uri should refer to thing/resource(noun) insted of action(verb). HTTP already defines actions like get, post,...
		forward slash (/) to indicate hierarchical relationships
		Do not use trailing forward slash (/) in URIs
		Use hyphens (-) to improve the readability of URIs- if resource name has multiple words
		Use lowercase letters
		Do not use file exyensions - it looks bad doesnt add any advantages removing them make URIs short
		Never use crud function name in URIs because http methods already indicate actions
		Use query component to filter URI collection
			when you want to filter. sort, or search a collection use query parameter insted of created new URi	/api/products?page=1&limit=10
