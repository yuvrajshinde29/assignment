—--------------------------------------------------------
MongoDB is a NoSQL database
Store data in documents(similar to JSON objects)

Each object is called Document.
Documents are grouped into CollectionsCollections are stored in database.

Hierarchy
	MongoDB Server
 	└── Database (e.g. myAppDB)
      		└── Collection (table)
        		   	└── Documents ( row)
				 └── Fields (key–value pairs)
Why mongo:
	Schema less : no fixed structure. You can have flexible fields in each documents.
JSON like storage:
	Easy to work with Node.js and JavaScript 
Scalable: 
	Built for large data and distributed systems.
Fast read writes:
	Optimized for performance using BJSON format(binary JSON)
	
	BSON: data in binary format,
		Machine readable,
		Support more datat ypes than json,
		Fast parsing and encoding
	Limitations of BSON: 16 MB document size limit,
				Binary, not human-readable,
				Slightly larger than json because BSON stores extra metadata for each field

		Note: In MongoDB, a collection is not actually created until it gets content!
		
	---------------
	2. insert documents
		
		I) inserOne()		
			db.collection_name.insertOne(object)
		II) insert many documents
			db.collection_name.inserMany(array of objects/documents)
	
		16 MB per document.
	--------------
	3. find data: to select data from collection
			
			db.collection.find(query, projection)
				this method accepts query object if left empty return all documents
				
				db.collection_name.find();
				
			db.collection.findOne(query, projection): 
				return fisrt matched document
				b.collection_name.findOne();	
				
			Parameters: query and projection
			
				I) query :
					filter like where      	 				
				----------
				II) Projection:
					controls which fields to include in result.
					if omitted returns all fields 
					
					{ fieldName: 1 } // include field
					{ fieldName: 0 } // exclude field
					
					You can’t mix inclusion and exclusion in the same projection (except _id).
					db.users.find({}, { name: 1, password: 0 })	//--error
					--
				pretty print (format output)
					db.user.find().pretty()
		---------
		update document:
			updateOne(filter,update) or updateMany()
			
				update one : updates first matching document.
					db.users.updateOne(
						{ name: "abc" },
						 { $set: { age: 22, city: "Goa" } },
					)
					
			updateMany(): update all matching documents
					db.users.updateMany(
						{ name: "abc" },
						{ $set: { age: 26 } }
					)
					output:
						{
							acknowledged: true,
							insertedId: null,
							matchedCount: 1,
							modifiedCount: 1,
							upsertedCount: 0
						}
						
			replaceOne(): Replaces entire existing document (not partial update)
				replaceOne(filter, replacementDocument)
				
			upsert option update if matching document found else create new one
				db.users.updateOne(
					{ name: "David" },
					{ $set: { age: 22, city: "Goa" } },
					{ upsert: true }  	//----
				)
		-----------
		update in arrays
		-----------
		Delete Document:
			deleteOne() or deleteMany()
			
			deleteOne() : delete first document that matches the filter.
				db.users.deleteOne({ city: "Pune" })
			
			deleteMany(): delete all matching documents
				db.users.deleteMany({ city: "Pune" })
			
			findOneAndDelete(): delete first matching document and returns document
		
			db.collection_name.drop(): Deletes the collection and all its indexes.
