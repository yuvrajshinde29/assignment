		1. Method 1 - we can define model using 
			sequelize.define();
		
				const User = sequelize.define("User", {
						// Column definitions
						id: {
							type: DataTypes.INTEGER,
							primaryKey: true,
							autoIncrement: true,
						},
						username: {
							type: DataTypes.STRING,
							allowNull: false,
						},

							type: DataTypes.STRING,

					}

		2. Method 2 — Extending the Model Class
		
		const { Model, DataTypes } = require("sequelize");
		const sequelize = require("../config/db");
			
		class User extends Model {
			// Instance method
			getDisplayName() {
				return this.name.toUpperCase();
			}
			
			// Static method
			static findByEmail(email) {
				return this.findOne({ where: { email } });
			}
		}
			
		User.init(
			{
				name: {
				type: DataTypes.STRING,
				allowNull: false,
				},
				email: {
				type: DataTypes.STRING,
				unique: true,
				},
			},
			{
				sequelize,   // connection instance
				modelName: "User",
				tableName: "users",
				timestamps: true,
			}
		);
			
		  module.exports = User;
			
			
		-----------------
			columnName: {
			type: DataTypes.<TYPE>,    // data type (required)
			allowNull: true,           // whether NULL values allowed
			defaultValue: <value>,     // default if none provided
			unique: true,              // unique constraint
			primaryKey: true,          // makes column primary key
			autoIncrement: true,       // auto-increment for integers
			field: "custom_column",    // custom column name in DB
			comment: "optional note",  // adds comment in DB
			validate: { ... },         // validation rules
			references: { ... },       // foreign key
			}
			{
			freezeTableName: true, //  prevents pluralization of table name in postgress(each model represent one record but table represent many)
			tableName: 'Employees	// we can provide table namedirectly
			}

			|  Part                            | Description                                                  |
			| -------------------------------- | ------------------------------------------------------------ |
			|  `sequelize.sync()`              | Converts Sequelize models → SQL tables                       |
			|  sequelize.sync({ force: true }) | Drops and recreates tables each time                         |
			|  sequelize.sync({ alter: true }) | Updates tables safely                                        |
			|  `require("./models/...")`       | Must be done before `.sync()` so Sequelize knows your models |

Data Types:

	STRING :
		DataTypes.STRING
	
	BOOLEAN:
		DataTypes.BOOLEAN; // TINYINT(1)
		
	NUMBER:
		DataTypes.INTEGER; // INTEGER
		DataTypes.BIGINT; // BIGINT
		DataTypes.BIGINT(11); // BIGINT(11)
		
		DataTypes.FLOAT; // FLOAT
		DataTypes.FLOAT(11); // FLOAT(11)
		DataTypes.FLOAT(11, 10); // FLOAT(11,10)
		
		DataTypes.REAL; // REAL            PostgreSQL only.
		DataTypes.REAL(11); // REAL(11)        PostgreSQL only.
		DataTypes.REAL(11, 12); // REAL(11,12)     PostgreSQL only.
		
		DataTypes.DOUBLE; // DOUBLE
		DataTypes.DOUBLE(11); // DOUBLE(11)
		DataTypes.DOUBLE(11, 10); // DOUBLE(11,10)
		
		DataTypes.DECIMAL; // DECIMAL
		DataTypes.DECIMAL(10, 2); // DECIMAL(10,2)

	Date:
		DataTypes.DATE; // DATETIME for mysql / sqlite, TIMESTAMP WITH TIME ZONE for postgres
		DataTypes.DATE(6); // DATETIME(6) for mysql 5.6.4+. Fractional seconds support with up to 6 digits of precision
		DataTypes.DATEONLY; // DATE without time
		
	UUID:
		A UUID (Universally Unique Identifier) is a 128-bit value used to uniquely identify a record
		
		{
			type: DataTypes.UUID,
			defaultValue: DataTypes.UUIDV4 // Or DataTypes.UUIDV1
		}
		
			| Version | Sequelize Constant | Description                           |
			| ------- | ------------------ | ------------------------------------- |
			| v1      | `DataTypes.UUIDV1` | Time-based UUID                       |
			| v4      | `DataTypes.UUIDV4` | Randomly generated UUID (most common) |

Model Instances:

	#Creating an Instance:
		although model is class you cannot create instance by using new operator  directly.
		
		-build():
			create a js object(instance of model)
			do not interact with db yet
			triggers no sql queries.
			(synchronous method)
			Ex-
				const jane = User.build({ name: 'Jane' });
				console.log(jane.name); // "Jane"
				
		-save():
			to save the instance in database save method used.
			(asynchronous method)
			Ex-
				await jane.save();
				console.log('Jane was saved to the database!');
		-----------------------------------------------
		-create() - (build() + save() in one step)
		
			const xyz = User.create({name : 'xyz'})
			console.log(xyz.name)		//xyz
	--------------------------------------------------------------------------------
	Note: logging instances:
		const jane = await User.create({ name: 'Jane' });
		// console.log(jane); // Don't do this this will print unwanted extra log
		console.log(jane.toJSON()); // This is good!
	--------------------------------------------------------------------------------
	#Updating an instance:
		if you change value of some field of an instance , calling save will update accordingly
		
		const jane = await User.create({ name: 'Jane' });
		console.log(jane.name); // "Jane"
		jane.name = 'Ada';						//-----------------------------------
		// the name is still "Jane" in the database
		await jane.save();
		// Now the name was updated to "Ada" in the database!
		
		
		-set() :you can update multiple fields at once using set() method
		
			const jane = await User.create({ name: 'Jane' });

			jane.set({
			  name: 'Ada',
			  favoriteColor: 'blue',			//----------------------------------
			});
			// As above, the database still has "Jane" and "green"
			await jane.save();
			// The database now has "Ada" and "blue" for name and favorite color
		-update specific set of fields 
			const jane = User.create({name: "jane"})
			await jane.update({name : "update_name"})
			--or--
			await User.update(
					{ name: 'Alice Updated' },  // new values
					{ 
						where: { id: 1 },         // condition
						returning: true           // Postgres only: return updated rows
					}
				);
					
			
			| Method             | Updates Immediately?  | Which Fields Update?         | Typical Use Case         |
			| :----------------- | :---------------------| :--------------------------- | :----------------------- |
			| `.update({ ... })` | ✅ Yes                | Only fields in the argument  | Quick partial updates    |
			| `.save()`          | ✅ Yes                | All changed fields in memory | Save all pending changes |

	#Incrementing and decrementing integer values:
		- for an incrementand and decrement value of instance without running into concurrency issue sequelize provide increment() and decrement() instance methods.
		
	
		const jane = await User.create({ name: 'Jane', age: 100 });
		const incrementResult = await jane.increment('age', { by: 2 });
		// Note: to increment by 1 you can omit the `by` option and just do `user.increment('age')`
		
		// In PostgreSQL, `incrementResult` will be the updated user, unless the option
		// `{ returning: false }` was set (and then it will be undefined).
		
		// In other dialects, `incrementResult` will be undefined. If you need the updated instance, you will have to call `user.reload()`.
		
		
		You can also increment multiple fields at once:
		
			const jane = await User.create({ name: 'Jane', age: 100, cash: 5000 });
			await jane.increment({
			age: 2,
			cash: 100,
			});
			
			// If the values are incremented by the same amount, you can use this other syntax as well:
			await jane.increment(['age', 'cash'], { by: 2 });

	Operators:
		const { Op } = require("sequelize");
		Post.findAll({
		where: {
			[Op.and]: [{ a: 5 }, { b: 6 }],            // (a = 5) AND (b = 6)
			[Op.or]: [{ a: 5 }, { b: 6 }],             // (a = 5) OR (b = 6)
			someAttribute: {
			// Basics
			[Op.eq]: 3,                              // = 3
			[Op.ne]: 20,                             // != 20
			[Op.is]: null,                           // IS NULL
			[Op.not]: true,                          // IS NOT TRUE
			[Op.or]: [5, 6],                         // (someAttribute = 5) OR (someAttribute = 6)
		
			// Using dialect specific column identifiers (PG in the following example):
			[Op.col]: 'user.organization_id',        // = "user"."organization_id"
		
			// Number comparisons
			[Op.gt]: 6,                              // > 6
			[Op.gte]: 6,                             // >= 6
			[Op.lt]: 10,                             // < 10
			[Op.lte]: 10,                            // <= 10
			[Op.between]: [6, 10],                   // BETWEEN 6 AND 10
			[Op.notBetween]: [11, 15],               // NOT BETWEEN 11 AND 15
		
			// Other operators
		
			[Op.all]: sequelize.literal('SELECT 1'), // > ALL (SELECT 1)
		
			[Op.in]: [1, 2],                         // IN [1, 2]
			[Op.notIn]: [1, 2],                      // NOT IN [1, 2]
		
			[Op.like]: '%hat',                       // LIKE '%hat'
			[Op.notLike]: '%hat',                    // NOT LIKE '%hat'
			[Op.startsWith]: 'hat',                  // LIKE 'hat%'
			[Op.endsWith]: 'hat',                    // LIKE '%hat'
			[Op.substring]: 'hat',                   // LIKE '%hat%'
			[Op.iLike]: '%hat',                      // ILIKE '%hat' (case insensitive) (PG only)
			[Op.notILike]: '%hat',                   // NOT ILIKE '%hat'  (PG only)
			[Op.regexp]: '^[h|a|t]',                 // REGEXP/~ '^[h|a|t]' (MySQL/PG only)
			[Op.notRegexp]: '^[h|a|t]',              // NOT REGEXP/!~ '^[h|a|t]' (MySQL/PG only)
			[Op.iRegexp]: '^[h|a|t]',                // ~* '^[h|a|t]' (PG only)
			[Op.notIRegexp]: '^[h|a|t]',             // !~* '^[h|a|t]' (PG only)
		
			[Op.any]: [2, 3],                        // ANY (ARRAY[2, 3]::INTEGER[]) (PG only)
			[Op.match]: Sequelize.fn('to_tsquery', 'fat & rat') // match text search for strings 'fat' and 'rat' (PG only)
		
			// In Postgres, Op.like/Op.iLike/Op.notLike can be combined to Op.any:
			[Op.like]: { [Op.any]: ['cat', 'hat'] }  // LIKE ANY (ARRAY['cat', 'hat'])
		
			// There are more postgres-only range operators, see below
			}

Getters, Setters & Virtuals:
	
	sequelize allows you to define custom getter and setter functions for model attributes.
	they let you manipulate data when reading or writing to model.
	-------------------------------------------------------------------------
	Getter : 
		getter let you control how an attribute value is returned
			A getter is a get() function defined for one column in the model definition.
			model.getDataValue("email") --gets raw value from database 

			email: {
				type: DataTypes.TEXT,
				get() {
					let rawValue = this.getDataValue("email");  //if we do this.email it will again call get(recurssion infinite loop)
					return rawValue ? rawValue.toUpperCase() : null;
				},
			},
			
	------------------------------------------------------
	Setter:
		A setter lets you control how an attribute’s value is stored (written).
				
			this.setDataValue(fieldName, value) --set a field’s raw value directly

			password: {
				type: DataTypes.STRING,
				set(value) {
				// Hash password before saving
				this.setDataValue("password", `hashed_${value}`);
				},
			},
		Useful for hashing passwords, formatting names, etc.
	------------------------------------------------------
	Virtual fields:
		Virtuals are attributes not stored in the database, but computed from other fields.
		
		Defined using DataTypes.VIRTUAL.
					
			const Product = sequelize.define("product", {
				name: DataTypes.STRING,
				price: DataTypes.FLOAT,
				tax: DataTypes.FLOAT,
				totalPrice: {
					type: DataTypes.VIRTUAL,
					get() {
					return this.price + this.tax;
					},
				},
			});
			// totalPrice doesn’t exist in the database table.
			// It’s dynamically calculated when accessed.

Validations & Constraints: both ensure data is clean and consistant.
	
	-Validations:
		validation check preformed at sequelize level (done in JavaScript before saving to DB)
		
		name: {
			type: DataTypes.STRING,
			allowNull: false,
			validate: {
				notEmpty: true, // cannot be ''
				len: [2, 50],   // between 2 and 50 chars
			}
		},
		name: {
			type: DataTypes.STRING,
			allowNull: false,               // Field is required
			validate: {
			notEmpty: { msg: "Name cannot be empty" },
			len: {
				args: [2, 50],
				msg: "Name must be between 2 and 50 characters long"
			}
			}
		},
		email: {
			type: DataTypes.STRING,
			allowNull: false,               // Required field
			unique: true,
			validate: {
			isEmail: { msg: "Email must be a valid email address" }
			}
		},
		
			
		-built in validator

Paranoid:
	 paranoid tables perform a soft-deletion of records, instead of a hard-deletion
			
		to make Model paranoid pass the paranoid: true option to model defination	
		-at time of delete Sequelize just sets a timestamp in a special column called deletedAt
			
		Example:
			
			const { DataTypes } = require('sequelize');
			const sequelize = require('../config/db');
				
			const User = sequelize.define('user', {
				name: DataTypes.STRING,
				}, {
				paranoid: true,        // enables soft delete
				timestamps: true,      // required for paranoid
				deletedAt: 'deletedAt' // optional custom column name
			});
				
			when you call	
				await User.destroy({ where: { id: 1 } });

			Sequelize run this SQL
				UPDATE "users" SET "deletedAt" = '2025-10-14 12:00:00' WHERE "id" = 1;

			The record stays in the database, but Sequelize will ignore it in queries (like findAll, findOne, etc.) automatically.

		Querying with Paranoid Models
			1. const users = await User.findAll(); 	//will not include rows where deletedAt is not null.
			
			2. inclued soft deleted records
				const data = User.findAll({paranoid:false})	//return all records with deleted records
			3. restore deleted Records
				await User.restore({where : {id : 1}});		//set deletedAt back to null
			| Option / Method                 | Purpose                                     |
			| ------------------------------- | ------------------------------------------- |
			| `paranoid: true`                | Enables soft delete                         |
			| `deletedAt`                     | Timestamp column used for deletion tracking |
			| `.destroy()`                    | Marks row as deleted by setting `deletedAt` |
			| `.findAll()`                    | Excludes soft-deleted rows                  |
			| `.findAll({ paranoid: false })` | Includes deleted rows                       |
			| `.restore()`                    | Undo soft delete (sets `deletedAt = NULL`)  |

	Types of Associations:
		------------------------------------------------
		1. One-to-one(hasOne/belongsTo):
			A one-to-one relationship means:
				One row in Table A is associated with exactly one row in Table B, and vice versa.
				Usually, you use a foreign key in one of the tables to reference the other.
			
			Sequelize provide two methods for one-to-one association
				hasOne() → Source model “has one” target model
				belongsTo() → Target model “belongs to” source model

			Example:
				const { DataTypes } = require('sequelize');
				const sequelize = require('./db');
					
				// User model
				const User = sequelize.define('User', {
					name: DataTypes.STRING,
				});
					
				// Profile model
				const Profile = sequelize.define('Profile', {
					bio: DataTypes.TEXT,
					avatar: DataTypes.STRING,
				});
					
				// Associations
				User.hasOne(Profile, {
					foreignKey: 'userId',   // foreign key in Profile table
					as: 'profile',           // optional, alias for association
				});
					
				Profile.belongsTo(User, {
					foreignKey: 'userId',   // same foreign key
					as: 'user',
				});
				------
				How It Works
					User.hasOne(Profile) → Sequelize knows Profile table stores userId.
					Profile.belongsTo(User) → Sequelize can join Profile → User easily.
					This sets up userId in Profile table automatically
				------
				// Get user with profile
				const userWithProfile = await User.findOne({
					where: { id: 1 },
					include: { model: Profile, as: 'profile' },
				});
				
				// Get profile with user
				const profileWithUser = await Profile.findOne({
					where: { id: 1 },
					include: { model: User, as: 'user' },
				});
				-----
				Key Notes
					The table that has the foreign key is the one that belongsTo.
					as is optional but recommended for clarity. but also need to include in include:{} and must be same 
					You can also use onDelete: 'CASCADE' if you want the profile to be deleted automatically when the user is deleted.	
					
				foreignKey → affects database column
				as → affects JavaScript property name and query include
		--------------------------------------------
		2. one-to-many relationship:

			One record in Model A can have many records in Model B.
			
			
			model1.hasMany(model2);
			model2.belongsTo(model1);
			
			
			-----
			// User Model
			const User = sequelize.define('User', {
			name: DataTypes.STRING,
			});
			
			// Post Model
			const Post = sequelize.define('Post', {
			title: DataTypes.STRING,
			content: DataTypes.TEXT,
			});
			
			----
			// Association: One user has many posts
				User.hasMany(Post, { foreignKey: 'userId' }); // optional foreignKey name
				Post.belongsTo(User, { foreignKey: 'userId' });
				
			 // Creating data
				const user = await User.create({ name: 'Alice' });
				await Post.create({ title: 'First Post', content: 'Hello world!', userId: user.id });
				await Post.create({ title: 'Second Post', content: 'Another post', userId: user.id });
							
			// Fetch user with posts
				const result = await User.findOne({ 
					where: { id: user.id },
					include: Post
				});
				
				console.log(JSON.stringify(result, null, 2));

		Many-to-Many relationship :

			One record of Table A can be associated with many records of Table B,
			and
			One record of Table B can be associated with many records of Table A.
		
		Sequelize uses a junction table (also called a through table) to handle Many-to-Many relationships.
			A.belongsToMany(B, { through: 'JunctionTableName' });
			B.belongsToMany(A, { through: 'JunctionTableName' });

				through can be:
					A string (Sequelize will create the table automatically)
					Or a Sequelize model (if you want to customize the junction table)
			--------------------------------------
			
			// Models
			const Student = sequelize.define('Student', {
				name: DataTypes.STRING
			});
			
			const Course = sequelize.define('Course', {
				title: DataTypes.STRING
			});
			
			// Junction table (optional, can store extra info)
			const StudentCourses = sequelize.define('StudentCourses', {
			enrollmentDate: {
				type: DataTypes.DATE,
				defaultValue: Sequelize.NOW
			}
			});
			
			// Many-to-Many association
			Student.belongsToMany(Course, { through: StudentCourses });
			Course.belongsToMany(Student, { through: StudentCourses });
			
			 // Create Students
			const yuvraj = await Student.create({ name: 'Yuvraj' });
			const abhi = await Student.create({ name: 'Abhi' });
			
			// Create Courses
			const nodeCourse = await Course.create({ title: 'Node.js Basics' });
			const sequelizeCourse = await Course.create({ title: 'Sequelize Advanced' });
			
			// Enroll Students in Courses
			await yuvraj.addCourses([nodeCourse, sequelizeCourse]); // Yuvraj in both courses
			await abhi.addCourse(sequelizeCourse);                  // Abhi only in Sequelize
			
			// Show enrolled courses for each student
			const students = await Student.findAll({ include: Course });
			console.log(JSON.stringify(students, null, 2));

	-------------------------------------------
	Creating associations referencing a field which is not the primary key
		it is possible using foreignKey and targetKey
		
		By default, Sequelize assumes:
		The source model's primary key (id) is referenced by the foreign key in the target model.
		Example: User.hasOne(Profile) → Profile.userId references User.id.
		
		But if you want to reference a different column, you need to explicitly specify:
		foreignKey → the column in the target model
		targetKey → the column in the source model that you want to reference (non-PK)

		Example:
			Suppose we have a User model where username is unique but not the primary key:

			const User = sequelize.define('User', {
				id: { type: DataTypes.INTEGER, primaryKey: true, autoIncrement: true },
				username: { type: DataTypes.STRING, unique: true },
			});
				
			const Profile = sequelize.define('Profile', {
				bio: DataTypes.STRING,
				user_username: DataTypes.STRING, // foreign key pointing to User.username
			});
				--
			User.hasOne(Profile, {
				foreignKey: 'user_username', // column in Profile
				sourceKey: 'username',        // column in User to reference must unique if not primary key
			});
				
			Profile.belongsTo(User, {
				foreignKey: 'user_username',
				targetKey: 'username',
			});
				
			sourceKey in hasOne/hasMany → the column in the source model to link.
			targetKey in belongsTo → the column in the target model to link.
			foreignKey → the column in the model that holds the reference.

	#Connection pool: 
		
		-Pooling = reuse of DB connections instead of creating new ones every time/for each query
		
		it can configured via the pool option in sequelize constructor.
			
		const { Sequelize } = require('sequelize');
		
		const sequelize = new Sequelize('database', 'username', 'password', {
			host: 'localhost',
			dialect: 'postgres', // or mysql, mariadb, sqlite, mssql
			pool: {
				max: 10,       // Maximum number of connections in pool
				min: 2,        // Minimum number of connections in pool
				acquire: 30000, // Max time (ms) to try to get a connection before throwing error
				idle: 10000    // Max time (ms) a connection can be idle before being released
			}
		});

		| Option    | Description                                                          |
		| --------- | -------------------------------------------------------------------- |
		| `max`     | Maximum connections allowed in the pool                              |
		| `min`     | Minimum connections maintained in the pool                           |
		| `acquire` | Time Sequelize will try to get a connection before throwing an error |
		| `idle`    | Time a connection can remain idle before being released              |
		
	How it works internally

		Sequelize maintains a queue of available connections (min initially created).
		When a query is executed:
		If a connection is available → use it.
		If none are free but max is not reached → create a new connection.
		If max is reached → wait up to acquire ms for a free connection.
		After query execution → connection returns to pool.
		Idle connections exceeding idle ms are closed.
	
	Benefits
		Performance: Faster queries because connections are reused.
		Scalability: Efficiently handles multiple concurrent queries.
		Resource management: Prevents too many open connections to DB.

	Indexes:
		indexes are used to improve database query performance by allowing faster lookups on one or more columns.
		
		define them in the indexes array when defining model
		get created at sync()

			const User = sequelize.define("User", {
			
				email: {
					type: DataTypes.STRING,
					allowNull: false,
				},
				
			}, {
				indexes: [
				{
					unique: true, // unique index
					fields: ['email']
				},
				{
					fields: ['username'] // normal index
				},
				{
					name: 'email_username_index', // custom index name
					fields: ['email', 'username'] // composite index
				}
				]
			});

		Index Options:
			unique: Ensures no duplicate values for the field(s).
			fields: Array of columns included in the index.
			name: Optional custom name for the index.
			using: Type of index. Examples:
				BTREE (default)
				HASH
						using: 'BTREE'
		-------
		Column-level Unique Constraint (alternative to index):
			email:{
				type: DataTypes.STRING,
				unique: true, // create unique index automatically
				allowNull: false, 
			}
		-----
		Composite indexes:
			index combined two or more col/attributes
			
			create if frequently quering by multiple columns together

			{
				indexes:[
					{
						fields:['firstName','lastName']
					}
				]
			}
		----
		Notes:
			Indexes speed up reads but slow down writes (insert/update/delete) because the index also needs updating.
			Always create indexes on columns used in WHERE, JOIN, ORDER BY clauses.
			Sequelize sync() will create the indexes in the database when force: true or alter: true is used.
	----------------------------------------------------------------------------------------------------------------------------------------------------	Migrations:
		just like version control systems such as git use to manage changes in code, you use migrations to keep track of change in database
		
		transfer database in one state to another state and vice versa.
		Those state transition are saved in migration files which describes how to get to new state and revert back 
	
		migration in Sequelize is basically a javaScript file that tells Sequelize :
			-what to change in database (e.g create table, add/remove column)
			-and how to undo that change(rollback)
			
		Each migration has two main function:
			-up:  runs when you apply(migrate) changes
			-down: runs when you revert(undo) these these changes
		programer define them manually but dont call manually, they will automatically call by CLI

Transactions:
	two ways of using transactions:
		1. Managed Transactions: 
			sequelize will automatically rollback the transaction if error is thrown or commit the transaction otherwise.
			try{
				await sequelize.transaction(async(t)=>{
				
					const user = User.create({userName:'abc'},{transaction: t})
					
					const user = Profile.create({user[userId]},{transaction: t})
					
					//if any thing inside thows, it auto rollback 
				})
				//if all gooes good then the transaction is commited
			}catch(){
				console.log('transaction rollbacked')
			}
		2. Unmanaged Transactions:
			commiting and rollback the transaction done manually by user 
	
				//1. start a transaction from your connection and save it to variable
				
				const t = sequelize.transaction();
				
				try{
					const user = User.create({userName:'abc'},{transaction: t})
					
					const user = Profile.create({user[userId]},{transaction: t})
					
					await t.commit() //manually cummit
					console.log('transaction cummited')
					
				}catch(error){
					await t.rollback()	//manually rollback
					console.log('transaction rollbacked')
				}

hooks:
	hooks are functions that run specific points in a model's lifecycle 
	such as before or after creating, updating, deleting, validating, or quering records.
	
	use:
		-Automatically hash password before saving in db
		-log acions(before delete, after update)
		-validate or modify data before saving
		
	Declaring hooks:
		three ways to declare hook:
		
		1. inside Model.init or sequelize.define()
			const User = sequelize.define('User',{
				name:DataTypes.STRING,
				email:DataTypes.STRING,
				
			},{
				hook:{
					beforeCreate: (user,options)=>{
						console.log('beforeCreate hook: ',user.name)
					}
					afterCreate: (user,options)=>{
						console.log('afetrCreate hook: ',user.name)
					}
				}
			})
	
		2. using model.addHook()
		
			User.addHook('beforeCreate',()=>{
				console.log('beforeCreate hook: ',user.name)
			})

			User.addHook('afterCreate',()=>{
				console.log('afterCreate hook: ',user.name)
			})
		3. Using sequelize.addHook() --global hooks
			sequelize.addHook('beforeValidate', (instance, options) => {
				console.log('Validating instance of', instance.constructor.name);
			});
		--------------------------------------------------------
		Removing hook:
			Only hook with name param can be removed.
			can have many hooks with same name.
			
		User.addHook('beforeCreate','hookName',(user, options)=>{
		
		})
		
		User.removeHook('beforeCreate','hookName');

		------------------------
		connection hooks:
			sequelize.beforeConnect(async (config) => {
				console.log('Connecting to DB...');
			});
			
			sequelize.afterConnect((connection, config) => {
				console.log('Connected to DB ');
			});
			
			sequelize.beforeDisconnect(() => {
				console.log('Disconnecting...');
			});
			
			sequelize.afterDisconnect(() => {
				console.log('Disconnected ');
			});
